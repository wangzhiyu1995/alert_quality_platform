# 告警指标梳理文档

## 1. 指标概览

当前系统中支持按**域、子域、业务系统、规则ID**粒度筛选和统计的指标共有8个：

| 指标名称 | 接口路径 | 指标类型 |
|---------|---------|---------|
| 夜间事件数 | /api/incidents/new-metrics | 计数指标 |
| 变更期间事件数 | /api/incidents/new-metrics | 计数指标 |
| 无效告警率 | /api/incidents/new-metrics | 比率指标 |
| 自动升级率 | /api/incidents/new-metrics | 比率指标 |
| 告警抖动率 | /api/incidents/new-metrics | 比率指标 |
| 手册覆盖率 | /api/incidents/rule-coverage | 覆盖率指标 |
| 场景关联率 | /api/incidents/rule-coverage | 覆盖率指标 |

## 2. 各指标计算逻辑

### 2.1 夜间事件数 (night_count)
**计算逻辑**：
- 单个规则粒度：统计该规则在夜间窗口（默认22:00-07:00）内触发的告警事件数
- 域/子域/系统粒度：统计该维度下所有规则在夜间窗口内触发的告警事件总数

**实现方式**：遍历事件，检查每个事件的创建时间是否在夜间窗口内

### 2.2 变更期间事件数 (change_count)
**计算逻辑**：
- 单个规则粒度：统计该规则在变更窗口（默认22:00-23:00）内触发的告警事件数
- 域/子域/系统粒度：统计该维度下所有规则在变更窗口内触发的告警事件总数

**实现方式**：遍历事件，检查每个事件的创建时间是否在变更窗口内

### 2.3 无效告警率 (invalid_rate)
**计算逻辑**：
- 单个规则粒度：该规则的无效告警数 / 该规则的总告警数 × 100%
- 域/子域/系统粒度：该维度下的无效告警数 / 该维度下的总告警数 × 100%

**实现方式**：根据配置的无效告警规则（关闭方式为auto、关闭耗时<5分钟、认领次数<1）判断每个事件是否为无效告警

### 2.4 自动升级率 (escalation_rate)
**计算逻辑**：
- 单个规则粒度：该规则中自动升级次数>0的告警事件数 / 该规则的总告警数 × 100%
- 域/子域/系统粒度：该维度下自动升级次数>0的告警事件数 / 该维度下的总告警数 × 100%

**实现方式**：检查每个事件的timeout_escalations字段是否大于0

### 2.5 告警抖动率 (jitter_rate)
**计算逻辑**：
- 单个规则粒度：该规则中属于抖动的告警事件数 / 该规则的总告警数 × 100%
- 域/子域/系统粒度：该维度下属于抖动的告警事件数 / 该维度下的总告警数 × 100%

**特殊说明**：无论按域、子域还是系统筛选，都会**先按规则ID分组**，再统计每个规则内的抖动事件。只有同一规则触发的告警，在30分钟内有2条或以上相邻触发的才会被标记为抖动事件。

**实现方式**：
1. 先根据筛选条件获取告警事件集合
2. **按规则ID分组**告警事件（核心步骤）
3. 对每个规则的事件按触发时间排序
4. 检查**同一规则内**的相邻事件是否在抖动窗口内（默认30分钟）
5. 标记相邻的抖动事件
6. 统计所有规则的抖动事件数和总事件数
7. 计算抖动率

### 2.6 手册覆盖率 (runbook_coverage)
**计算逻辑**：
- 单个规则粒度：该规则是否有处理手册（0%或100%）
- 域/子域/系统粒度：该维度下有处理手册的规则数 / 该维度下的总规则数 × 100%

**实现方式**：
1. 提取筛选条件下所有唯一的规则ID
2. 查询规则表中对应规则的runbook_url字段
3. 计算有值记录占总记录的百分比

### 2.7 场景关联率 (scene_coverage)
**计算逻辑**：
- 单个规则粒度：该规则是否有关联场景（0%或100%）
- 域/子域/系统粒度：该维度下有关联场景的规则数 / 该维度下的总规则数 × 100%

**实现方式**：
1. 提取筛选条件下所有唯一的规则ID
2. 查询规则表中对应规则的scene字段
3. 计算非空记录占总记录的百分比

## 3. 计算逻辑一致性分析

### 3.1 计数类指标（夜间事件数、变更期间事件数）
- **单个规则粒度**：统计该规则符合条件的事件数
- **域/子域/系统粒度**：统计该维度下所有规则符合条件的事件总数
- **一致性**：✅ 完全一致，都是直接聚合统计

### 3.2 比率类指标（无效告警率、自动升级率、告警抖动率）
- **单个规则粒度**：该规则的目标事件数 / 该规则的总事件数 × 100%
- **域/子域/系统粒度**：该维度下的目标事件数 / 该维度下的总事件数 × 100%
- **一致性**：✅ 完全一致，都是先筛选出符合条件的事件集合，再计算目标事件占比

### 3.3 覆盖率类指标（手册覆盖率、场景关联率）
- **单个规则粒度**：二元判断（0%或100%）
- **域/子域/系统粒度**：有属性的规则数 / 总规则数 × 100%
- **一致性**：✅ 基本一致，都是基于规则维度的统计，但单个规则粒度表现为二元结果

## 4. 筛选机制分析

所有指标均支持以下筛选条件：
- 业务域 (domain)
- 业务子域 (sub_domain)
- 业务系统 (system)
- 规则ID (rule_id)
- 时间范围 (start_date, end_date)

筛选机制的工作原理：
1. 根据筛选条件构建查询，获取符合条件的告警事件集合
2. 基于该集合进行指标计算
3. 不同粒度的筛选通过调整查询条件实现

## 5. 数据来源与同步

### 5.1 告警事件数据
- 来源：直接从incidents表获取
- 更新机制：实时更新

### 5.2 规则维度数据
- 来源：从alert_rules表获取，数据来源于告警事件
- 更新机制：在告警事件导入时自动同步更新
- 扩展字段：runbook_url和scene字段预留了扩展支持

## 6. 可配置项

| 指标 | 可配置项 | 配置方式 |
|------|----------|----------|
| 夜间事件数 | 夜间窗口时间范围 | TimeWindowConfig表（window_name='night'） |
| 变更期间事件数 | 变更窗口时间范围 | TimeWindowConfig表（window_name='change'） |
| 无效告警率 | 无效告警判断规则 | InvalidAlertRule表 |
| 告警抖动率 | 抖动时间窗口（分钟） | TimeWindowConfig表（window_name='jitter'） |

## 7. 结论

当前系统中的指标在不同粒度（域、子域、系统、规则）下的计算逻辑**保持一致**：

1. **计数类指标**：直接聚合统计符合条件的事件数
2. **比率类指标**：目标事件数 / 总事件数 × 100%
3. **覆盖率类指标**：有属性的规则数 / 总规则数 × 100%

所有指标均支持统一的筛选机制，可灵活按不同维度进行统计分析。
